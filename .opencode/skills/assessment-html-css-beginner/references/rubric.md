# Rubric

Total score: 100

## 1. HTML structure and basic semantics, 20

Check:
- Whether the page has a clear structure
- Whether tags are basically reasonable
- Whether the page is not just random nested divs

Suggested scoring:
- 16-20: clear structure, semantics basically reasonable
- 10-15: understandable structure, average tag usage
- 0-9: chaotic structure

## 2. CSS layout ability, 25

Check:
- Whether layout is stable
- Whether flex or grid is used
- Whether layout is not mainly hard-positioned with absolute

Suggested scoring:
- 20-25: clear and stable layout
- 12-19: usable but average
- 0-11: obvious layout issues

## 3. Styling completeness, 20

Check:
- Whether cards, buttons, lists, inputs have basic styles
- Whether hover/focus exist
- Whether the page has basic visual hierarchy

Suggested scoring:
- 16-20: styling is complete
- 10-15: usable but plain
- 0-9: very little styling

## 4. Responsive adaptation, 20

Check:
- Whether desktop is normal
- Whether mobile remains readable
- Whether there is obvious overlap, overflow, or breakage

Suggested scoring:
- 16-20: desktop and mobile both acceptable
- 10-15: some adaptation exists
- 0-9: almost no mobile handling

## 5. Code quality, 10

Check:
- Whether file structure is clear
- Whether HTML/CSS are readable
- Whether there is no heavy mechanical repetition

Suggested scoring:
- 8-10: clear and readable
- 5-7: understandable but average
- 0-4: very messy

## 6. README quality, 5

Check:
- Whether the student explains structure
- Whether layout choices are described
- Whether responsive handling is described

Suggested scoring:
- 5: clear
- 3-4: basic but short
- 0-2: almost absent

## Major deductions

- Page cannot open or core files are broken
- Obvious complete template copy
- No mobile adaptation at all
- README severely perfunctory
