## 考核说明

本次开发方向考核共包含 **5 道题**，每题 **100 分**，总分 **500 分**。

- **第 1-2 题** 主要考察前端基础能力，包括页面结构理解、基础交互实现与简单功能开发

每道题都配有详细的评分细则，具体要求可查看各题对应仓库中的文档。

建议大家优先尝试：

这两题属于开发方向中较为基础的内容，整体难度适中，适合先熟悉做题方式，也方便在正式推进前补充相关知识、掌握基本开发流程。

## 提交方式

所有题目都必须通过 **`dev-2026-grading` 仓库的 Issue 模板**提交。

不接受：

- 私聊提交
- 随意发仓库链接
- 其他非 Issue 形式的提交

提交时请注意：

1. 先完成你自己的题目仓库
2. **不要修改题目仓库名**
3. 回到评分仓库的 `Issues`
4. 使用提供的模板创建提交 Issue
5. 填写：
   - 你的 GitHub 用户名
   - 题目仓库名
   - 你的提交仓库地址
   - 备注（如果有未完成部分，可以提前说明）

## 额外说明

开发方向更强调：

- 实际工程能力

- 功能完整性
- 逻辑清晰性
- 良好的开发习惯

建议大家循序渐进，逐步完成全部题目。
