# Rubric

Total score: 100

## 1. DOM selection and event binding, 20

Check:
- Whether core DOM nodes are selected correctly
- Whether load, click, submit, and filter events are bound as needed

Suggested scoring:
- 16-20: core DOM and events are complete
- 10-15: mostly complete
- 0-9: major missing bindings

## 2. GET request and list rendering, 25

Check:
- Whether the task list is requested correctly
- Whether returned data is rendered to the page
- Whether rendering structure is basically correct

Suggested scoring:
- 20-25: request and rendering both work well
- 12-19: basic completion but average
- 0-11: major missing logic

## 3. POST request and submission handling, 20

Check:
- Whether form data is read correctly
- Whether POST payload is sent correctly
- Whether success result updates the page or gives feedback

Suggested scoring:
- 16-20: submission flow is complete
- 10-15: mostly complete
- 0-9: major submission issues

## 4. Data handling and filtering, 15

Check:
- Whether status filtering works
- Whether page data stays basically correct after interaction

Suggested scoring:
- 12-15: filtering works well
- 7-11: partial completion
- 0-6: missing or broken

## 5. Error handling and user feedback, 10

Check:
- Whether failed requests show basic feedback
- Whether submit success/failure gives basic feedback

Suggested scoring:
- 8-10: feedback is clear
- 5-7: some feedback exists
- 0-4: almost none

## 6. Code quality and README, 10

Check:
- Whether main.js is basically readable
- Whether README explains the solution briefly

Suggested scoring:
- 8-10: readable and explained
- 5-7: acceptable
- 0-4: messy or undocumented
